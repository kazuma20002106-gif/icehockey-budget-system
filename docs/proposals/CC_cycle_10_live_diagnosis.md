# 提案: 自動起動プロンプトは「検証対象値を事前計算してテンプレート同梱」を標準化

作成者: CC  
作成日: 2026-06-24  
背景: Cycle 10 Phase 2 実機テストで実Claudeが `cc.done.json` に `revision="revision_2"` / `result="sandbox_compliant"` と自然言語的に書き、Maestro Runner の厳密検証でPAUSEした。

## 提案

自動起動CCに渡すプロンプトでは、Maestro Runner側が**厳密一致で検証する値**について、以下を標準パターンとする。

1. **Runner側で事前計算できる値は計算してプロンプトに直接埋め込む**
   - 例: `source_p1_sha256` はP1ファイルから事前に算出し、プロンプトに値そのものを記載。
2. **完成形JSONテンプレートを同梱する**
   - `<...>` プレースホルダだけ自分で埋めれば完成する形を示す。
3. **禁止例を明記する**
   - `revision_2`・`sandbox_compliant` のような「やりがちな逸脱」を具体例として禁止と書く。

## 効果

- 実Claudeの「自然言語的な値ゆらぎ」による無駄なPAUSE（＝課金消費）を抑制できる。
- 「安全装置は正しく止めるが、正常系も通る」状態に近づく。

## 留意

- 安全装置（Runner側の厳密検証）は**緩めない**。あくまでプロンプト側で正解を提示するだけ。
- 事前計算値（P1ハッシュ等）の埋め込みは、git安全監査・P1存在チェックを通過した後に行う（無言離脱を防ぐため）。
