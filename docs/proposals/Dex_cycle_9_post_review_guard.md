# Dex提案 — 危険タスク越権実装時の事後レビューゲート

## 提案

Airや他AIが危険タスクを誤って直接実装してしまった場合、事後レビューではコード差分だけでなく、作業ツリー全体の混入ファイル確認を必須にする。

## 背景

今回、Excel金額計算ロジック本体は概ね正しかったが、以下の副産物が残っていた。

- `replace.py`
- `test.py`
- `src/main/resources/templates/test.py`
- 無関係なマニュアル差分/文字化け疑い

危険タスクでは、正しいコード変更だけでなく「余計なものを混ぜないこと」も安全性の一部。

## 追加したいチェック

P3/P4テンプレートに以下を追加する。

- `git status --short` の結果
- 未追跡ファイル一覧と扱い
- resources配下へ一時ファイルが混入していないこと
- 無関係なマニュアル差分がコミット対象外であること
