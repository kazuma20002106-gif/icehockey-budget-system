# ＋αの提案：共通同期後のプロジェクト固有補強ルール

共通テンプレ同期は便利だが、同期後に `AGENTS.md`, `CLAUDE.md`, `.cursorrules` が薄くなりすぎるリスクがある。

次回以降は、同期スクリプトに「共通テンプレを配った後、各プロジェクト専用の追記ブロックを末尾に合成する」仕組みを入れるとよい。

例:

- 共通部分: `AI共通マニュアルテンプレ`
- アイスホッケー専用追記: `docs/manual_overrides/icehockey.md`
- 同期結果: 共通部分 + 専用追記

これにより、共通ルールの一元管理と、プロジェクト固有の強い安全ルールを両立できる。
