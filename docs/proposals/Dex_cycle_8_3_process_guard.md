# Dex Cycle 8.3 プラスアルファ提案: 完了時ガードの明文化

## 提案

Dex/Air/CC共通で、作業完了前に必ず見る「出口チェック」を `AGENTS.md` または `CURRENT_STATUS.md` に短い定型として追加する。

## 反映状況

`AGENTS.md` に以下を追加済み。

- 新チャット・コンテキスト圧縮時の起動ルール
- 作業終了前の出口ゲート
- Kazumaxの「引継ぎもお願い」をトリガーに、次担当へのコピペ文へ起動ルール文を同梱するルール

## 背景

今回Dexは、技術レビュー内容は作成できたが、以下の運用が漏れた。

- 長文レビューを最初から裏ファイルに保存すること。
- チャットを短い合図文に留めること。
- 次担当へのコピペ文を出すこと。
- プラスアルファ提案をチャットと `docs/proposals/` の両方に保存すること。

原因は、作業対象ファイルの読解を優先し、完了時ルールを自分のチェックリストとして扱えなかったこと。

## 追加したい出口チェック

```text
作業終了前チェック:
1. 本文が10行を超える、または次担当が読む内容なら docs/handoff/ に保存したか。
2. チャットは「現在地」「次担当」「読むファイル」「コピペ合図文」だけになっているか。
3. プラスアルファ提案がある場合、チャットと docs/proposals/ の両方に同じ内容を出したか。
4. P4 OKの場合、詳細チェックリストをP4に保存し、短縮版をチャットに出したか。
5. CURRENT_STATUS.md の更新が必要な作業か確認したか。
```

## 期待効果

チャットが変わっても、各AIが同じ出口動作を再現しやすくなる。
特に「レビュー内容は正しいが、引き継ぎ形式が崩れる」事故を減らせる。
