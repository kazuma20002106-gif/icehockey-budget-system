# ＋α提案：年度末テンプレートの未使用シート整理

Cycle 12Aで`src/main/resources/書類.xlsx`を本物原本（111シート）へ完全置換しました。年度末出力（Cycle 12A）で実際に使用するのは以下の5シート＋動的生成分のみです。

- 様式２－１
- 様式２－２
- 様式２－２－１　事業別決算書（選手強化費／トップチーム活用／ふるさと）
- 既存の2-4/2-5/2-6テンプレート（クローン元）

残りの約106シートは、過去の①〜⑰事業の実施報告・参加者名簿・領収書など、実在する氏名・電話番号・メールアドレスを含む実データです。出力ブックには含まれません（未使用シートは出力前に削除されるため）が、アプリの配布物（JAR）自体には常に同梱される形になります。

## リスク

- 個人情報（氏名・連絡先）が、実際には不要な形でアプリの配布物に残り続ける。
- テンプレートファイルサイズが103KB→約2.7MBに増加し、エクスポート毎に不要なシートも含めてメモリに読み込まれる。

## 提案

Cycle 12B/12C、またはそれ以降の別サイクルで、以下のいずれかを検討してください。

- A案: 使用する5シート＋2-4/2-5/2-6テンプレートのみを残した、個人情報を含まない軽量版テンプレートを作成する。
- B案: 現状維持（内部利用ツールのため個人情報同梱のリスクは限定的、と判断する）。

いずれの場合も、シート削除は数式参照（様式2-2の2-2-1参照など）を壊さないよう、Dexの事前レビューを挟むことを推奨します。
