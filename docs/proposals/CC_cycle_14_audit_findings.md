# Cycle 14監査で発見した事項：`/activity`一覧の支出合計と様式2-2系の総合計は、個人雑費がある場合に構造的に一致しない

## 結論

**バグではなく、様式2-2に「個人雑費」の行が存在しないことに起因する、設計上の既定の差異**と判断する。ただし「合算が正常に行われているか」を最優先とする方針に照らし、Kazumaxの認識と一致しているかを確認しておきたいため報告する。

## 症状

1参加者以上に個人雑費（`Expense.miscellaneousCost`）が入力されている事業がある場合、以下の2つの「支出合計」が一致しなくなる。

- `/activity`（活動一覧）の支出合計列・グランドトータル：個人雑費を**含む**
- legacy `/export`の様式2-2プレビュー・Excel（`ExportController.preview()` / `ExcelExportService.exportForm22Summary()`）の総合計：個人雑費を**含まない**
- 年度末決算ファイル出力（`/export/year/preview` / `/export/year/download`）の様式2-2・2-2-1・2-3の総合計：個人雑費を**含まない**

## 再現手順・実測値（Cycle 14監査でJDBC経由の一時テストデータにより確認）

一時的に作成したテストデータ（監査完了後に削除済み）:

- 事業「Cycle14監査テスト(成年男子)」（2026年度・選手強化費・成年男子）
  - 参加者A: Expense 2件（交通費計2,500円、宿泊費計5,000円、個人雑費計300円）
  - 参加者B: Expense 1件（交通費500円、宿泊費1,000円、個人雑費0円）
  - 参加者C: Expense 1件（全項目NULL）
  - 事業サマリ: 借用料1,000／需用費2,000／駐車料金300／報償費400／役務費500／旅行雑費(単価700×3人×2日=4,200)

実測結果:

| 画面 | 総合計 |
|---|---|
| `/activity` 支出合計（この事業単体） | 17,700円（個人雑費300円を含む） |
| legacy `/export` 様式2-2プレビュー・Excel（この事業のみ選択） | 17,400円（個人雑費300円を含まない） |
| 年度末決算プレビュー 様式2-2-1（選手強化費、この事業のみが対象年度・区分に該当する条件） | 17,400円相当（内訳合算で個人雑費を含まない） |

差額は常に「対象事業の個人雑費合計」と一致する。

## 原因（コード確認）

- `ActivityController.list(...)` の `expenseTotal` は、参加者ごとの `Expense` を全件ループし `transportCost + accommodationCost + miscellaneousCost` を加算している（個人雑費を含む）。
- `ExportController.preview()` の `exportType=="2-2"` 分岐、および `ExcelExportService.populate22Summary(...)` は、`transportCost` と `accommodationCost` のみを合算し、`miscellaneousCost` を合算式に含めていない（個人雑費を含まない）。
- 年度末決算（`ExcelExportService.buildAnnualPreview(...)` / `exportAnnualClosingBook(...)`）も同様に、様式2-2/2-2-1の集計は個人雑費を含まない。
- これは実装ミスというより、**様式2-2・2-2-1・2-3という公式帳票そのものに「個人雑費」という費目の欄が存在しない**ことに起因すると考えられる（`書類.xlsx`のセル構成にも個人雑費専用セルは無い）。

## 影響

- 実データ（2026年度、既存8事業）では個人雑費がすべて0円だったため、Cycle 12/13の検証時にはこの差異が顕在化しなかった。
- 今後、担当者が活動記録の「雑費」欄（個人ごとのその他経費）に金額を入力した場合、`/activity`の支出合計と、実際に提出するExcel決算書（様式2-2系）の総合計が一致しなくなる。
- 決算書自体の金額（Excel出力）は、様式2-2の定義に沿っている限り正しい。ただし「`/activity`の合計を見て検算する」運用をしている場合、両者が食い違って見えるため、混乱や「合算が間違っているのでは」という誤解の原因になり得る。

## 提案する対応方針（CCでは未修正。次サイクルでの判断を仰ぐ）

以下のいずれかの方針を、Air/Dex/Kazumaxで決めていただきたい。

1. **現状維持**: 個人雑費は様式2-2に欄が無い費目であり、`/activity`の合計は「事業に紐づく全経費（帳票に出ない分も含む）」を示す別の指標として明示する。`/activity`画面に注記（例:「※様式2-2の総合計には個人雑費は含まれません」）を追加する案。
2. **`/activity`側を様式2-2に合わせる**: `/activity`の支出合計から個人雑費を除外し、様式2-2系の総合計と完全一致させる。ただし、個人雑費を含む「本当の意味での事業支出合計」を別途どこかに表示する必要が生じる可能性がある。
3. **様式2-2側に個人雑費を合算する**: 様式2-2・2-2-1・2-3の集計に個人雑費を追加する。ただし公式帳票のセル構成・行定義を変更することになるため、原本Excelとの整合性を別途監査する必要がある（危険度が高い）。

いずれの方針も金額計算・帳票出力に関わる危険タスクのため、採用する場合はAir計画→Dex事前レビュー→CC実装→Dex事後レビューの完全プロセスを適用すること。

## 参考: 監査で確認できたその他の項目（バグなし）

- 複数Expense（1参加者2件）は、交通費・宿泊費・個人雑費のいずれも正しく全件合算された。
- 全項目NULLのExpenseは、合計を壊さずに0円として扱われた。
- 事業サマリが存在しない事業（`project_summary_expenses`行なし）でも、旅行雑費・借用料等はエラーなく0円扱いされた。
- 少年男子など、既存データに無かったカテゴリを新規に追加しても、様式2-2-1・2-3の当該カテゴリ行に正しく分離されて計上され、他カテゴリの合計を汚染しなかった。
- 旅行雑費（事業サマリ側）と個人雑費（参加者ごと）は、コード上完全に独立した変数・独立した表示ラベルで扱われており、混同は確認されなかった。
