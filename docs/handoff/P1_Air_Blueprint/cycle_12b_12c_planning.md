# Cycle 12B / 12C 実装設計（Blueprint）

CCが実装中の 12A には干渉せず、残りのフェーズの設計をまとめました。

## 1. Cycle 12B: 予算管理と様式2-3連動

### 1-1. `budget_allocations` テーブルのスキーマ
プロジェクトごとに設定される「対象区分（成年男子など）」単位で予算を保持します。
- `id` (INT, PK)
- `budget_type_id` (INT, 1:選手強化, 2:トップ, 3:ふるさと)
- `target_category` (VARCHAR, 例: '成年男子')
- `fiscal_year` (INT)
- `allocated_amount` (INT, 内示額)
- `created_at` (TIMESTAMP)

### 1-2. 予算管理画面のUI
- **場所**: サイドバーに「予算管理」を追加
- **機能**: 
  - 画面上部で「年度」を選択（デフォルトは今年度）
  - 各補助金区分ごとに、DBに存在する `target_category` の一覧を表示
  - それぞれの「内示額」をテキストボックスで入力し、「保存」ボタンで一括保存

### 1-3. 様式2-3（変更実績報告書）のセル座標と出力ロジック
原本のCSV解析結果に基づく設計です。
- **内示額の列**: `K列`
- **移動後の総額（決算額）の列**: `T列`

**【重要: 行の動的検索】**
原本テンプレートのカテゴリ行（「例）少年男子」など）は固定ではないため、Java側（Apache POI）で **「B列の文字列（種別名）」を上から検索し、一致した行のK列とT列に金額を書き込む** 動的検索ロジックを採用します。（ハードコードによるズレを防ぐため）

## 2. Cycle 12C: 大容量プレビューと出力UI

### 2-1. 出力パラメータ入力ダイアログ
- **トリガー**: 「年度末決算ファイル出力」ボタン押下時

### 2-2. 大容量プレビューのタブ化UI

- **タブ構成**: 
  - `[表紙(2-1)]` `[まとめ(2-2)]` `[選手強化(2-2-1)]` `[トップ(2-2-1)]` `[ふるさと(2-2-1)]` `[変更報告(2-3)]`
- **データ取得**: `/api/export/preview-yearly` エンドポイントを新設し、各シートのHTMLテーブルをJSON形式で一括返却。JavaScriptでタブ切り替え時に表示を切り替える。

## 3. 12Bと12Cの分割について
**結論：明確に分割すべきです。**
- **理由**: 12Bは「DB設計、金額集計ロジック、Excelの動的検索書き込み」という**バックエンド中心の危険タスク**です。一方、12Cは「JavaScript、モーダル、タブUI」という**フロントエンド中心のタスク**です。
- これを混ぜると、万が一バグが出た際に「Excelの書き込みがおかしいのか、プレビューの表示がおかしいのか」の切り分けが難しくなります。CCには「まず12Bを完成させ、Excelが正しく出力されることを確認（コミット）してから、12CのUI実装に進む」よう指示します。

## 4. 危険ポイント（Dexの事前監査が必要な箇所）
12Bにおける以下のポイントは、実装前にDex(P2)の目視チェックが必須です。
- **様式2-3の動的行検索ロジック**: 文字列が完全一致しない場合（スペースの有無など）のフォールバック処理。
- **移動額の自動計算**: 上部（17行目付近）にある「種別間変更」の要約欄について、システムがどこまで自動計算するか（※Kazumax様への確認事項に付随）。

## 5. Kazumax様との合意事項（運用方針の決定）

以下の2点について、Kazumax様と運用方針を合意しました。CCはこれに従って実装してください。

1. **様式2-3上部の「変更理由・移動額」欄**
   - **決定**: **システムでは自動計算せず、出力後のExcelで手入力する運用とします。**
   - **理由**: システムで下部の「内示額・決算額」だけを正確に出力すれば十分であり、上部の要約は人間の手で柔軟に記載した方が確実なため。

2. **予算管理画面のカテゴリ表示**
   - **決定**: **「すでに事業として登録された実績があるカテゴリ」だけを自動で一覧表示して入力させる方式（B案）とします。**
   - **理由**: マスター登録画面を作る手間を省き、運用をシンプルにするため。年度末の実績報告時にはすでに事業が存在しているため、この方式で問題ありません。
