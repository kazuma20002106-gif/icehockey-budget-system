# Cycle 12: 年度末決算ファイル一括出力 実装草案 Take2

Dexの監査（cycle_12_original_formula_audit.md）にて判明した、セル座標のズレ、直値（ハードコード）の存在、およびテンプレート不一致を修正したTake2の設計図です。

## 1. テンプレートファイルの方針
現在アプリ内にある `src/main/resources/書類.xlsx` は情報が古く、トップチーム用のシートなどが欠損しています。
そのため、本実装を開始する前に、Kazumax様から提供された本物原本 `②【報告様式】R7-宮崎国スポ強化戦略プロジェクト補助金 (1).xlsx` をアプリの `src/main/resources/書類.xlsx` として**完全置換（移植）**します。
これによりトップチーム用2-2-1シートも利用可能になります。

## 2. 外部参照（リンク）の排除方針
本物原本に残存している外部ファイルへの参照（`\\192.168.145.12\...`）は、アプリに移植する際、または出力処理の段階で**すべて削除・値渡しに変換、もしくはエラーにならないよう内部参照に修正**します。これにより、出力されたExcelを開いたときの「リンクの更新」エラーを防ぎます。

## 3. 様式2-2（まとめシート）の直値（ハードコード）修正方針
Dexの監査で指摘された以下の「数式が組まれておらず直値になっているセル」については、**Javaコードから値を直接書き込むのではなく、アプリに取り込むテンプレート（書類.xlsx）側のセルをあらかじめ正しい数式に修正しておく**アプローチをとります。

- `様式2-2` の `AE17` (ふるさと交通費): `='様式２－２－１　事業別決算書（ふるさと）'!J16` に修正
- `様式2-2` の `AE19` (ふるさと宿泊費): `='様式２－２－１　事業別決算書（ふるさと）'!J18` に修正
- `様式2-2` の `AE21` (ふるさと雑費): `='様式２－２－１　事業別決算書（ふるさと）'!J20` に修正
- `様式2-2` の `J27` (需用費合計): `=SUM(Q27:AK28)` に修正（※またはQ27+X27+AE27等、原本の他の合計セルと同じロジックに修正）

これにより「2-2-1をJavaで埋めれば、2-2はExcelの数式で完全自動計算される」という当初の美しい設計を維持します。

## 4. 各様式の正確なセル座標マッピング（修正版）

### A. 様式2-1（表紙）
Dexの指摘を受け、実際の座標を使用します。
- 提出日（年）: `AF3`
- 提出日（月）: `AJ3`
- 提出日（日）: `AN3`
  （※AA3やAG3等は元号や「月」「日」などのラベルの可能性があるため、実装時にApache POIで付近のセル構造を確認し、最適なセルへ書き込む）
- 団体名: `AB9`
- 代表者職・氏名: `AB11`
- 対象年度: `E15`

### B. 様式2-3（変更実績報告書）
Dexの指摘を受け、実使用領域（1〜34行のK列等）を使用します。
- 選手強化費 内示額: `K17`
- 成年男子 内示額: `K25`
- ふるさと成年男子 内示額: `K33`
※詳細な行は実装時にExcelExportService内で再度実ファイルを参照し、正確な行へ書き込みます。「O85等」の推測座標は使用しません。

### C. 2-2-1（事業別決算書）
- **選手強化費**: `J16`(交通), `J18`(宿泊), `J20`(雑費), `J22`(駐車), `J24`(借上), `J26`(報償), `J28`(需用), `J30`(役務)
- **トップチーム**: `J16`(交通), `J18`(宿泊), `J22`(借上), `J24`(報償), `J28`(需用), `J30`(役務) ※原本通り存在するため出力対象に含める
- **ふるさと**: `J16`(交通), `J18`(宿泊), `J20`(雑費), `J28`(需用), `J30`(役務)

## 6. 実装フェーズの分割（Dex推奨案を採用）
危険リスクを最小化するため、実装は以下の3段階（Cycle 12A / 12B / 12C）に分割して行います。
1. **Cycle 12A**: テンプレートの完全置換、直値・外部リンクの修正、2-1/2-2/2-2-1の一括出力ロジック実装（既存2-4〜2-6との統合）
2. **Cycle 12B**: `budget_allocations` テーブル追加、予算管理画面実装、様式2-3への出力
3. **Cycle 12C**: 大容量プレビューのタブ化、出力ダイアログUIの仕上げ

## 7. 検証データ（テストケース）
- 選手強化費のみ / ふるさとのみ / トップチームを含む場合（成年男子・女子などが混在）
- 各費目（交通費〜役務費）がそれぞれ0円/1円以上の場合
- 参加者0人の事業が存在する場合
上記パターンで正しくExcelが出力され、様式2-2の合計がピッタリ合うかを検証します。
