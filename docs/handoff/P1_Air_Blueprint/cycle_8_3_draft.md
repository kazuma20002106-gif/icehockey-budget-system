## 📝 Proposed Changes (CC向け 実装指示)

### 1. Excelシート名生成と事業グループ化（ExcelExportService.java）
- 複数事業を出力する際、シート名を `[様式]_[補助金区分]_[種別]_[同種別内の順番①②③]` とする。（例：`2-4_選手強化費_成年男子_①`）
- **【バグ修正】** 補助金区分は「事業名文字列」から判定するのではなく、`project.getBudgetType().getName()` から正しく取得すること。
- **【グループ化】** 様式2-4で2つの事業を1シートに結合する際、同じ「種別（対象区分：成年男子等）」かつ「補助金区分」が同じもの同士でのみペアにする。
- **【採番とソート】**
  - 「補助金区分」＋「種別」ごとに独立して①から採番する。
  - 様式2-4で結合する際は、左側が古い日付（①）、右側が新しい日付（②）になるよう昇順ソートする。
  - 単独出力時にシート番号が偶数番になるバグを修正し、単独出力時は①固定等、正しく連番を振る。
- **【出力順序】** 一括出力時、`様式2-4全て → 様式2-5全て → 様式2-6全て` の順で出力し、その中で `成年男子 → 成年女子 → 少年男子 → 少年女子` の順にソートする。

### 2. 様式2-2-1（事業別決算書）のバグ修正（ExcelExportService.java）
- **【バグ修正】** 様式2-2-1の旅行雑費の集計において、旧仕様である「個人別雑費（expenses.miscellaneous_cost）の合計」を集計するのをやめる。
- 新仕様である「単価×人数×日数」（`projectSummaryExpense.getTravelMiscCost() * (人数) * projectSummaryExpense.getTravelMiscDays()` 等のロジック）で算出した金額を、種別ごとに正しく集計してセルへ出力するように修正する。

### 3. UI / UX改善（form.html / list.html / Controller）
- **宿泊設定・旅行雑費の複合入力（form.html）**
  - 宿泊数と単価、旅行雑費の日数と単価の入力枠を `input-group` を用いて合体させ、`[単価]円 × [日数/泊数]日/泊` の順に統一する。
  - 宿泊設定に `※単価は1泊分・最大3泊` を追記し、`col-md-4` や `text-nowrap` で改行崩れを防ぐ。
  - 旅行雑費を `[単価]円 × [日数]日 × [人数]人 = [合計]円` のフル計算式UIにし、JavaScript (`updateTravelMiscTotal`) で人数変更時にも自動計算されるよう実装。
- **宿泊費0円上書き防止の徹底（form.html）**
  - Cycle 8.2の「単価0入力時は既存値を保持する」ガード処理を維持すること。
  - 様式2-6明細行の「宿泊費」入力欄を `readonly` 化（グレー背景）し、基本情報の宿泊設定からの自動計算のみで値が入るよう強制する。
- **総合計レイアウトの刷新（form.html）**
  - `tfoot` の総合計表示を独立した新しい行（一段下）に分割し、`fs-5` で大きく表示する。
- **活動一覧の絞り込み（list.html / ActivityController）**
  - 絞り込み条件に「種別（対象区分）」のプルダウンと「事業名」の部分一致テキスト検索を追加する。
  - 絞り込み入力欄の配置順を「年度 → 月 → 補助金区分 → 種別 → 事業名」の順にする。
