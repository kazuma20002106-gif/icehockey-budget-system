# Cycle 14 Blueprint: 全体バグ監査およびリポジトリ健全化計画

Cycle 12（年度末出力）および Cycle 13（旅行雑費・複数Expense修正）という金額に関わる大型アップデートが完了したことを受け、システム全体の計算の一貫性と、AI開発環境（ワークスペース）の健全性を監査する計画です。
本サイクルでは新規機能の開発は行わず、**「金額の正確性の証明」と「負債の清算」**に特化します。

---

## 1. 金額・ロジック監査計画（デクスクルー連携推奨）

CCが手動テスト・コード確認を行う前に、Dexは以下の検証観点に基づく厳密なテストケースをCCへ指示してください。

### ① 複数Expenseの実データ検証
- **目的**: Cycle 13で修正した「`exList.get(0)` 縛りの解除」が正しく機能しているかの証明。
- **手法**: テストDB上で、特定の事業の1人の参加者に対し、意図的に複数の `Expense` レコード（例: 交通費2件、宿泊費2件）を登録する。
- **確認点**: 活動一覧画面、legacyプレビュー画面、年度末プレビュー画面、およびExcel出力結果のすべてにおいて、全件の合算値が正しく反映されていること。

### ② プレビュー表示とExcel出力の金額完全一致
- **目的**: 画面表示（ユーザーの確認用）と、実際に印字されるExcel（提出用）の数字に1円のズレもないことの証明。
- **対象導線**:
  1. **Legacy導線** (`/export` からの様式2-2, 2-4, 2-6 個別プレビューと出力)
  2. **新・年度末出力導線** (`/export/year/preview` と一括ダウンロード)
- **確認点**: 双方が全く同じデータを参照し、同じ合計値（総合計、交通費合計、宿泊費合計など）を算出していること。

### ③ 「旅行雑費」と「個人雑費」の混同リスク排除
- **目的**: 似た名称の変数が二重計上されたり、互いに入れ替わったりしていないかの確認。
- **確認点**: 参加者ごとの `miscellaneousCost` と、事業サマリの `travelMiscCost`（×人数×日数）が、それぞれの集計箇所で独立して正確に加算されていること。

## 2. 未追跡ファイルや古いHandoff混入リスクの棚卸し

Airが `git status` でワークスペースをスキャンした結果、`docs/handoff/` や `docs/proposals/` 配下に **数十個の未追跡ファイル（Cycle 8〜13の過去の指示書など）が大量に残留している** ことが判明しました。

### 🚨 放置するリスク
AIエージェント（特に新しく起動したCCなど）が、これらの古い仕様書やNGレビューの指摘を最新の指示と誤認し、**古いバグを再実装してしまう（AIハルシネーションの誘発）**危険性が極めて高い状態です。

### 🧹 健全化（クリーンアップ）指示
CCに対し、以下のリポジトリクリーンアップを実施させます。
1. **履歴のコミット**: Cycle 13までの必要なHandoff記録や提案書はすべて `git add` してコミットし、Gitの管理下に置く。
2. **不要ファイルの削除**: 一時的な確認に使われたファイル（例: `sheets_preview.txt`, `tmp_cycle12_check/`, `dummy_fail.md` 等）は完全に物理削除する。
3. **作業後確認**: `git status` が `working tree clean` になること。
