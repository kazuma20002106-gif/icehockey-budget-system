# Cycle 5: 超・改修サイクル（バグ修正・プレビュー・和暦変換）（P1 -> P2）

Dexへ。KazumaxのP5テストで大量のバグが発見されました。以下の8つの要件を満たすP2指示書（`docs/handoff/P2_Dex_to_CC/cycle_5.md`）を作成してください。これは非常に重要な修正サイクルです。

## ① プレビュー画面の導線修正
- **対象**: `src/main/resources/templates/activity/list.html`
- **内容**: 
  - 現在、活動一覧の「出力（まとめて、様式別）」ボタンを押すと直接Excelがダウンロードされる（`GET /activity/{id}/export`）。
  - これを廃止し、既存のプレビュー機能（`POST /export/preview`）へ遷移するようにフォームを修正する。
  - `form` タグを作成し、`projectIds` と `exportType`（"all", "2-4", "2-5", "2-6"）を送信してプレビュー画面を開く仕様にする。

## ② エクセルの和暦（令和）変換
- **対象**: `ExcelExportService.java`
- **内容**: 
  - 西暦を令和に変換するメソッド `getReiwaYear(int year)` を作成（西暦 - 2018）。
  - 様式2-2, 2-4, 2-5, 2-6 すべての出力処理において、年度のセルに書き込む値を「そのままの西暦」から「令和の数値」に変更する。

## ③ 様式2-4のバグ修正（重要）
- **対象**: `ExcelExportService.java`
- **内容**:
  - **既存図形の削除**: テンプレートに残っている手動記入用の黒い巨大丸（`HSSFSimpleShape` などの `Shape`）をプログラム実行時にすべて削除してから、新しい丸を描画する。
  - **合計金額の強制上書き**: テンプレートに入力されているダミー合計額（84050など）を消去し、システムが計算した正しい「計」をセルに書き込む。
  - **期日の和暦・曜日フォーマット**: 「期日」セルに出力する日付を `令和X年Y月Z日(曜)` の形式にする。（例: 令和7年9月27日(土)）
  - **相方自動連行システム**: 
    - 「1件の活動のみ（個別）」を出力する際、対象プロジェクトがその年度内で「何番目の活動か（日付順）」を判定する。
    - 奇数回目（1, 3, 5回目）なら左に印字し、右は空欄。
    - 偶数回目（2, 4, 6回目）なら右に印字し、左側には「1回前の活動（相方）」のデータをDBから取得して印字する。これにより常に2枚セットで出力されるようにする。

## ④ 様式2-5のバグ修正
- **対象**: `ExcelExportService.java`
- **内容**:
  - **操作ユーザーの印字**: テンプレート下部の「記入責任者氏名・電話番号」に、ユーザー設定（`UserSetting`）の氏名と電話番号を上書きする。
  - **事業実施日のフォーマット**: ヘッダーの【事業実施日】欄には `令和X年Y月Z日` と印字。もし参加者の中に宿泊を伴う者（泊数 > 0）がいれば、最大泊数を加算して `令和X年Y月Z日～W日` のように期間で印字する。
  - **宿泊対象者の「日付」と「〇」印字**: 参加者の「宿泊費」が1円以上の場合は、様式2-5の「宿泊対象者」の一番左の枠のヘッダー部（月 日 と書いてあるセル）に宿泊日（例: `1月24日`）を印字し、該当者の行には日付の代わりに「〇」を印字する。

## ⑤ 様式2-6のバグ修正
- **対象**: `ExcelExportService.java`
- **内容**:
  - **操作ユーザーの印字**: 様式2-6の「作成者名」および下部の「記入責任者氏名・電話番号」に、ユーザー設定の情報を上書きする。
  - **期日と受領日のフォーマット**: 「期日」と「受領日」のセルには西暦を含めず、`M/d`（例: `8/25`）の形式で印字する。見切れないようにフォーマットを整える。
  - **合計金額の強制上書き**: 様式2-6下部のダミー合計金額を消去し、印字した対象者の実際の合計金額を計算して書き込む。

## ⑥ 電話番号の自動ハイフン化と案内
- **対象**: `src/main/resources/templates/activity/layout.html`
- **内容**: 
  - 操作ユーザー設定モーダルの電話番号入力欄に `placeholder="例: 090-1234-5678"` を追加。
  - ヘルプテキストとして「※ハイフンなしで入力しても自動で補完されます」を記載。
  - JavaScriptで、電話番号入力時に自動でハイフンを挿入する処理を実装。

## ⑦ セレクトボックスの文字被り修正
- **対象**: `src/main/resources/static/css/style.css` 
- **内容**: `select.form-select` の右側パディング（`padding-right`）を大きくし、プルダウンの矢印が文字（「2026年度」など）と被らないように修正。

## ⑧ 活動一覧の月別フィルター機能
- **対象**: `ActivityController.java`, `list.html`, `ProjectMapper.java/xml`
- **内容**:
  - 現在の「年度」「補助金区分」に加えて「月別（4月, 5月... または 全て）」のセレクトボックスを追加し、絞り込み機能を実装する。

> **💡 Dex/CCへの重要なルール**
> 各様式の本来の書式（座標や文言）が分からなくなった場合は、必ずプロジェクト内にある原本ファイル（`書類.xlsx`）を開き、元のテンプレートフォーマットを参照・比較してから実装すること。

# ⚠️ Cycle 5.1 差し戻し＆高度化 要件（追加実装）
KazumaxのP5テストおよび追加ヒアリングにより、以下のバグ修正と大幅な高度化（神機能）を実装すること。

## ⑨ 様式2-4の丸（Ellipse）の座標修正
- **内容**: 「成年女子」や「強化合宿」などを囲む丸の座標が左上にずれている。`drawEllipse`の引数（`XSSFClientAnchor`）やオフセットを微調整し、文字に綺麗に被るように直すこと。

## ⑩ 様式2-4（右側）のダミーデータ完全消去
- **内容**: 個別出力時など、右側に相方がいない（`id2 == null`）場合、テンプレート右側のダミーデータ（人数4名、旅行雑費など）がそのまま出力されている。右側エリアのダミーデータをすべて白紙にする処理を徹底すること。

## ⑪ 補助金区分ごとの連番（丸数字）自動計算
- **内容**: その年度内のプロジェクトを日付順でソートし、**「補助金区分（FundingSource）」ごとに独立して**連番（1, 2, 3...）を計算する。この数値を丸数字（①、②、③...）に変換して使用する。

## ⑫ ヘッダーの丸数字動的書き換え
- **内容**: 
  - 様式2-4の左上セル（`様式２－４①②事業実施・実績報告書` など）を、実際の連番（例：③④など）を使って書き換える。
  - 様式2-5/2-6も同様に `様式２－５①` `様式２－６①` のように書き換える。

## ⑬ シートの命名ルール（シート名の重複回避と視認性）
- **内容**: 
  - シート名は内部IDではなく `[補助金名][様式名][連番]` とする。
  - 例：`選手強化2-4①②`、`選手強化2-5①`、`ふるさと2-4①②` など。
  - ※DB上のFundingSourceが「①選手強化費」のようになっている場合は、先頭の「①」や末尾の「費」などを適宜トリムするか、マッピングして見やすい名前にすること。

## ⑭ オール出力時の「様式別」シート並び替え（最強機能）
- **内容**: ユーザーが印刷後にそのまま綴じられるよう、出力シート群を以下の優先順位でソートする。
  1. 補助金区分（選手強化 → ふるさと等）
  2. 様式種別（2-4 → 2-5 → 2-6）
  3. 日付順（連番順）
  - つまり、`[選手強化の2-4全て]` → `[選手強化の2-5全て]` → `[選手強化の2-6全て]` → `[ふるさとの2-4全て]`... という順番になる。

## ⑮ 印刷範囲とA4サイズの強制（Fit to Page）
- **内容**: 全シート出力時、プログラム側でエクセルの「Fit to Page」設定を強制オン（`sheet.setAutobreaks(true)`, `sheet.getPrintSetup().setFitWidth((short)1)`, `sheet.getPrintSetup().setFitHeight((short)1)` 等）にする。
- 様式2-4については、青い点線を気にしなくて済むよう、最下部の「記入上の注意」までを含めた印刷エリア（`sheet.setPrintArea`）を明示的に設定し、A4・1ページにピタッと収めること。

# ⚠️ Cycle 5.2 再々差し戻し要件（最終調整）
KazumaxのP5再レビューにより、以下のさらなる調整が必要となったため、CCに修正させること。

## ⑯ 様式2-4の丸（Ellipse）の究極調整
- **内容**: 画像確認の結果、右側の項目（「遠征試合」や「成年女子」など）を囲む丸が、依然として左上にずれている。`dx1, dy1, dx2, dy2` などのアンカーオフセットをさらに（右方向・下方向へ）大きくプラス調整し、文字のど真ん中を囲むように完璧に微調整すること。

## ⑰ 様式2-6：「航空機」のデカ文字マージ出力
- **内容**: 様式2-6の交通区間において、交通手段が「航空機」などの場合（区間を示す「～」が含まれない場合）、上下2段に分割する処理を行わず、2段分のセル（例：行17と行18）を `CellRangeAddress` で結合（マージ）し、文字を中央揃えにして「航空機」と大きく（でかでかと）印字するように変更すること。

## ⑱ UI側：キロ数入力の制御（フロントエンド）
- **対象**: `activity.html` （または参加者入力モーダルのJS）
- **内容**: 
  1. 交通手段（Transport Method）のプルダウンの初期値（デフォルト）を「自家用車」にする。
  2. 交通手段が「自家用車」**以外**（電車、航空機、バスなど）に変更された場合、キロ数（Distance）の入力欄を「入力不可（disabled）」にし、すでに入力されていた場合は値をクリア（空）にすること。
  - 理由：自家用車以外（1km=17円の計算対象外）でキロ数が入力されると、エクセル上に「電車(90)km」といった無意味な印字がされてしまうため。
