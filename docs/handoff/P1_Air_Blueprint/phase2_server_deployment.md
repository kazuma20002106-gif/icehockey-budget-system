# フェーズ2 (本番サーバー構築) Blueprint

アイスホッケー予算管理システムの開発フェーズ（Cycle 1〜16）はすべて完了しました。
ここからは、**「依頼主がスマホやPCから24時間365日いつでも安全にアクセスして利用できる」**ようにするための本番環境構築（フェーズ2）に入ります。

---

## 1. サーバー環境の基本方針

以前ご相談いただいた通り、依頼主の「Windows 11 Home」を24時間稼働のサーバーとして使うのは非常に危険かつ不安定なため、**Linuxベースのクラウドサーバー（VPS）**を利用します。
（このVPSを一度借りてしまえば、次に予定している「居酒屋HPプロジェクト」のWebサーバーとしてもそのまま使い回すことができます！）

**【想定するシステム構成】**
- **OS**: Ubuntu Linux (22.04 LTS または 24.04 LTS)
- **Webサーバー**: Nginx (スマホからアクセスするための受付口)
- **アプリ**: Java 17 (Spring Boot 3)
- **DB**: MySQL 8.0
- **セキュリティ**: SSL化（https://〜 で始まる安全な通信）

---

## 2. Kazumax様への確認事項（VPSの準備）

サーバー構築作業は、まずKazumax様に「インターネット上のまっさらなサーバー（VPS）」をご用意いただくところからスタートします。

> [!IMPORTANT]
> **現在、LinuxのVPS（XServer VPS、ConoHa VPS、さくらのVPSなど）は既に契約済みでしょうか？**
> 
> - **[はい]**：契約済みの場合、サーバーのIPアドレス、初期ユーザー名（root等）、パスワード等がお手元にあるかご確認ください。（チャットには絶対にパスワードを書き込まないでください！）
> - **[いいえ]**：未契約の場合、私からおすすめのVPSサービスと契約手順（月額数百円〜千円程度）をご案内いたします。

---

## 3. 今後の作業ステップ（予定）

VPSがご用意でき次第、以下の手順で私（AIチーム）がコマンドや手順を一つずつ案内し、Kazumax様と一緒に構築を進めていきます。

1. **初期セキュリティ設定** (ハッカーに乗っ取られないためのファイアウォールやSSH設定)
2. **ミドルウェアのインストール** (Java 17, MySQL 8, Nginx)
3. **データベース(MySQL)の構築** (アイスホッケー用のDBとユーザー作成)
4. **アプリのビルドとアップロード** (現在完成しているシステムを`.jar`ファイルに固めてサーバーへ送る)
5. **常時起動設定** (サーバーが再起動しても自動でアプリが立ち上がるようにする)
6. **SSL化とドメイン設定** (スマホから安全にアクセスできるようにする)
