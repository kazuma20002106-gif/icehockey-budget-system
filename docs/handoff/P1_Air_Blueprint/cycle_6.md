## 🛠️ 修正の詳細（CCへの厳命）

### 1. `preview.html`：様式2-5（名簿）プレビューの5列化
**【事象】** CCが `v1.9.4` で5列にしたにも関わらず、`v1.9.5` で勝手に2列に戻してしまった。
**【修正】** 以下の5列構成に**再度変更**し、絶対に2列に戻さないこと。
- 列構成: `No` | `監督・選手別` | `氏名` | `年齢・学年` | `宿泊対象者`
- データがない項目（年齢など）はプレビュー上は空欄で構わないので、枠（列）は原本通りに必ず用意すること。

### 2. `preview.html` 及び `ExcelExportService.java`：期日と受領日のフォーマット
**【事象】** 「2026-06-10」のように西暦が含まれており、原本と異なる。
**【修正】** プレビュー画面（2-6明細表）および、Excel出力（2-6の該当セル）において、表示を `6/10` のような「月/日」形式（先頭のゼロ埋めなし `M/d` または `M/D`）に変換して出力すること。
- （例）`2026-06-10` → `6/10`

### 3. `ExcelExportService.java`：電話番号のハイフン表示
**【事象】** Excelの各シート下部に印字される「記入責任者」の電話番号からハイフンが消えている。
**【修正】** 
- データベースに保存されている電話番号文字列（`activeUser.getPhoneNumber()`）をそのまま印字すること。もしExcel書き込み時にハイフンを抜く処理があるなら削除する。
- もしハイフン無しで保存されているなら、出力時にハイフンを補完する。携帯（090-XXXX-XXXX）、固定（0985-XX-XXXXなど）に対応するのが難しければ、せめてユーザーが入力した生の文字列（ハイフン入り）がそのままExcelのセルに入るようにすること。

### 4. `layout.html` 及び `style.css`：サイドバーの亡霊（空白）完全除去
**【事象】** `mb-auto` は消え、`<ul>` も統合されたが、依然としてメニュー上部に巨大な空白が存在している。
**【修正】** 
- Bootstrapの固定ナビゲーション（`fixed-top`）とサイドバーのパディング（`pt-3` 等）の相性、または `top` プロパティのズレが原因。
- ブラウザのデベロッパーツールで見るのと同じようにコードを精査し、`v1.9.1`バッジの上、または「活動一覧」の上に不要な `padding` や `margin` （あるいは空のダミー要素）がないか確認し、左メニューが上部（黒いヘッダーのすぐ下）から綺麗に詰まって表示されるようにCSSとHTMLを修正すること。
