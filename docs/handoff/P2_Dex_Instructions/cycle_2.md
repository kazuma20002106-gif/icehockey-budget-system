# [C2: Dex(P2) ⇒ CC(P3)]

活動日入力フォーム（form.html）のUX・視認性改善を実施してください。
作業完了後、必ず pp.version=v1.8.3 への更新と .\mvnw compile を行い、srcとtargetの同期確認を忘れずに行うこと。

■ 対象ファイル
- src/main/resources/templates/activity/form.html
- src/main/resources/static/css/style.css (必要に応じて)

■ 実装要件
1. 事業の成果の初期値設定
   - project.projectOutcome の <textarea> から placeholder を削除。
   - 	h:text 等を用いて、値が空の場合は実文字で 技術力・チーム連携の向上 を初期表示させる。

2. テーブル入力欄の縮小サイズ除去
   - 様式2-5と2-6のテーブル内にある orm-control-sm, orm-select-sm, input-group-sm を全て除去し、標準サイズに戻す。

3. テーブルヘッダーの固定（Sticky化）
   - 両テーブルのヘッダー要素に対して position: sticky; top: 0; z-index: 2; を適用し、スクロール時も上部に固定する。（背景色が透けないよう注意）

4. 縦スクロール領域の高さを「ヘッダー＋4行」に最適化
   - 両テーブルの親divの max-height: 220px; を、入力欄標準化後の「ヘッダー1行＋4行分」がスクロール無しでぴったり見える高さ（概ね300px〜350px前後）に再調整する。

5. ツールチップの視認性改善
   - デフォルトの黒背景ツールチップを、CSSで「白系背景、黒文字、左揃え、フォントサイズ0.95rem等、最大幅350px等」に上書きし、改行が不格好にならない美しく読みやすいデザインに変更する。

6. 注釈テキストの不格好な改行修正
   - 日付欄の下にある「年度（4月始まり）は...」「※受領日は...」の <small> 要素が狭い幅で不自然に改行されている問題を、レイアウトや幅の調整で美しく直す。
