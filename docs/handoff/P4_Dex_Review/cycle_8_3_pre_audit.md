# Cycle 8.3 / 8.4 Dex事前監査レビュー

## 判定

Air草案 `docs/handoff/P1_Air_Blueprint/cycle_8_3_draft.md` は方向性OK。
ただし危険タスクのため、この草案のままCC直行はNG。
以下の補強を入れたうえでCCへ渡すこと。

## 修正必須

### 1. 補助金区分取得方法

草案では `project.getBudgetType().getName()` から補助金区分を取得する指示になっている。
しかし現行 `Project` モデルに `BudgetType budgetType` はなく、存在するのは `budgetTypeId`。
このまま実装するとコンパイルエラー、または不要なモデル/DB変更へ流れる危険がある。

CC向け指示では以下に修正すること。

- `ExcelExportService` 内に `budgetTypeLabel(Integer budgetTypeId)` helperを作る。
- `ActivityController#budgetLabel` と同じ対応にする。
- `project.getName()` から補助金区分を推測しない。
- 存在しない `getBudgetType()` を呼ばない。
- このためだけにDBスキーマ変更をしない。

### 2. 2-2-1旅行雑費セルの確定

旅行雑費の計算式は `summary.travelMiscCost * 参加人数 * summary.travelMiscDays` でよい。
ただし、出力セルが草案では曖昧。

CC向け指示では以下を必須にすること。

- 実装前に `書類.xlsx` の様式2-2-1を実読し、旅行雑費行を確認する。
- 出力セルと行番号根拠をP3報告に書く。
- セルが確定できない場合は推測で別行へ出力せず、P3で「旅行雑費セル未確定」と明記して止める。
- `Expense#getMiscellaneousCost()` を決算書の旅行雑費集計に使わない。

### 3. Excelシート名の安全条件

シート名形式は `[様式]_[補助金区分]_[種別]_[連番]` でよい。
ただしExcel制約が未記載。

CC向け指示では以下を追加すること。

- Excelシート名は31文字以内にする。
- `/ \ ? * [ ] :` など使用不可文字を安全な文字へ置換する。
- 同名衝突時は既存の `uniqueName` と同様に重複回避する。

### 4. ソート順の明確化

草案では種別順のみが強調されており、同一条件内の順序が不安定になり得る。

CC向け指示では一括出力時の各様式内ソート順を以下に固定すること。

1. 補助金区分ID昇順
2. 種別順: 成年男子、成年女子、少年男子、少年女子、その他
3. 活動日昇順
4. ID昇順

### 5. 2-4ペアリング

2-4の左右結合は同じグループ内だけで行う。

グループキー:

```text
budgetTypeId + targetCategory
```

ルール:

- グループごとに活動日昇順で並べる。
- 2件ずつペアにする。
- 左側は古い日付、右側は新しい日付。
- 1件だけ余った場合は左側に出力し、右側は `clearSide24` で空欄化する。
- 単独2-4出力は左側に対象、右側空欄で固定する。

### 6. UIコンパクト化の範囲

Cycle 8.4の「全html」は範囲が広すぎる。
Excel出力、金額計算、一覧検索と同時に全画面UI調整を行うとレビュー困難になる。

CC向け指示では以下を追加すること。

- 無関係な大規模リファクタは禁止。
- 対象テンプレートを明示する。
- 変更後に表示崩れがないか確認する。
