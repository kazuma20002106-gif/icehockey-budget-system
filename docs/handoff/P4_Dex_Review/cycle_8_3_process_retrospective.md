# Cycle 8.3 Dex運用ミス原因分析

## 結論

今回の問題は、チャットを変えたこと自体が主因ではない。
主因は、Dexが作業開始時に対象ドラフトと周辺レビューは読んだが、完了時の出口ルールとして `AGENTS.md` と `CURRENT_STATUS.md` をチェックリスト化しなかったこと。

その結果、以下が抜けた。

- 長いレビューを最初から裏ファイルに保存すること。
- チャット欄を短い合図文に留めること。
- 次担当へのコピペ文を出すこと。
- プラスアルファ提案をチャットと `docs/proposals/` の両方に出すこと。

## なぜ起きたか

### 1. 作業対象ファイルだけを優先して読んだ

Dexは `cycle_8_3_draft.md` と関連するP2/P4レビューを読み、技術レビューとしては必要な文脈を集めた。
一方で、作業完了時のチーム運用ルールを確認するための `CURRENT_STATUS.md` までは最初に読まなかった。

`CURRENT_STATUS.md` には、作業終了時に以下を行うルールが明記されていた。

1. `CURRENT_STATUS.md` を現在地に合わせて更新する。
2. 最終チャットに現在地サマリーとコピペ合図文を出す。
3. 合図文にはCycle/Take、最新P1/P3/P4のパス、次担当者が読むべきファイルを含める。

### 2. 事前監査を「コードレビュー回答」として処理してしまった

ユーザー依頼は「安全性・実装指示の明確さ・CCへ渡してよいかをレビュー」だった。
Dexはこれを通常のレビュー回答として扱い、結果をチャットに直接まとめた。

しかし、チーム運用上はこれは危険タスクのDex事前監査であり、P4レビューに近い成果物を `docs/handoff/` に保存するべきだった。

### 3. ルールが複数ファイルに分散していた

関連ルールは以下に分散している。

- `AGENTS.md`: 長文は裏ファイル、表チャットは合図文。プラスアルファ提案はチャットと提案箱の両方。
- `AI_TEAM_WORKFLOW.md`: 危険タスクはAir草案、Dex事前監査、CC、Dexレビューの流れ。
- `CURRENT_STATUS.md`: 作業完了時の現在地更新とコピペ合図文。
- `CLAUDE.md`: CC向けの完了報告テンプレート。

Dexは `AGENTS.md` と `AI_TEAM_WORKFLOW.md` の該当ルールを後から確認したため、初回回答で形式を外した。

### 4. プラスアルファ提案を「任意」と誤解した

`AGENTS.md` では、プロジェクト改善案やマニュアル更新案を出す場合、チャットと `docs/proposals/` の両方に保存することになっている。

今回の「チャットを変えても同じ動きにするには」という話は明確にマニュアル改善案なので、提案箱に保存すべきだった。
これも完了時チェックがなかったため漏れた。

## 対策

### Dexの作業開始チェック

チーム運用・引き継ぎ系の依頼では、最初に以下を読む。

1. `docs/handoff/CURRENT_STATUS.md`
2. `AGENTS.md`
3. 作業対象ファイル

技術レビューだけでなく、作業後の出力形式を先に確定する。

### Dexの作業終了チェック

最終回答前に以下を確認する。

1. レビュー本文が10行を超えるか。
2. 次担当に渡す内容か。
3. `docs/handoff/` に保存すべきか。
4. チャットは短い合図文だけになっているか。
5. プラスアルファ提案がある場合、同じ内容を `docs/proposals/` に保存したか。
6. `CURRENT_STATUS.md` 更新が必要な作業か。

### 出力の判断基準

- 技術判断が短く、次担当への正式引き継ぎでない: チャットのみ可。
- 10行超、複数論点、次担当が読むべき内容: `docs/handoff/` に保存し、チャットは短い合図文。
- マニュアル改善、運用改善、将来サイクルへの提案: `docs/proposals/` に同じ内容を保存し、チャットにも短く出す。
- P4 OKでKazumax実機確認に進む: P4ファイルに詳細チェックリスト、チャットに短縮チェックリスト。

## 今回の是正

- 事前監査レビューは `docs/handoff/P4_Dex_Review/cycle_8_3_pre_audit.md` に保存済み。
- 本原因分析は `docs/handoff/P4_Dex_Review/cycle_8_3_process_retrospective.md` に保存。
- プラスアルファ提案は `docs/proposals/Dex_cycle_8_3_process_guard.md` に保存。
- 根本対策として、`AGENTS.md` に「新チャット・コンテキスト圧縮時の起動ルール」と「作業終了前の出口ゲート」を追加済み。


```text
まず AGENTS.md と docs/handoff/CURRENT_STATUS.md を読んで、現在地・次担当・完了時ルールを確認してから作業して。
長文レビューや次担当への正式引き継ぎは docs/handoff/ に保存し、チャットは短い合図文だけにして。
プラスアルファ提案がある場合は docs/proposals/ にも同じ内容を保存して。
```
