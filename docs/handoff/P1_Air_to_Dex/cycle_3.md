# Cycle 3: ツールチップ化、デザイン修正、機能追加（P1 -> P2）

Dexへ。以下の5つの要件を満たすP2指示書（`docs/handoff/P2_Dex_to_CC/cycle_3.md`）を作成してください。

## ① 注意書きのツールチップ化（スマホ・タップ対応）
- `form.html` の「年度（4月始まり）」ラベルの下にある `small` タグのテキスト2行を削除し、ラベルの横に `<i class="bi bi-info-circle"></i>` を配置する。
- マウスオーバーだけでなく、スマホでのタップ（click/focus）でも表示されるよう、Bootstrapのツールチップ設定（`data-bs-toggle="tooltip"`、`data-bs-trigger="hover focus"` など）を適切に付与すること。

## ② ツールチップの視認性向上
- `style.css` にてBootstrapのツールチップ（`.tooltip-inner`, `.tooltip-arrow` 等）をオーバーライドする。
- 薄い灰色背景（例: `#f8f9fa`）、濃い枠線（例: `#6c757d`）、濃い文字色に変更して、ライトモードでも背景と同化しないようにする。

## ③ テーブルヘッダーの隙間修正（Sticky Headerバグ）
- 様式2-5、2-6のテーブルで、ヘッダー（`th`）がSticky指定されているが、スクロール時にヘッダーと中身の間に隙間が空く現象を修正する。
- `th` に対して `box-shadow` を使うか、`border-collapse` や `background-clip: padding-box;` などのCSSハックを用いて、隙間を完全に塞ぐこと。

## ④ 2-6交通費（距離×単価）の自動計算
- `form.html` の `recalculateTransportCosts()` は既にあるが、「距離（`.distance-input`）」や「単価（`#transportRate`）」を手入力した際に発火していない。
- `expenseBody` または対象の input に `input` イベントリスナーを追加し、ユーザーが手入力した瞬間に（距離 × 単価）が交通費欄に自動反映されるようにすること。

## ⑤ 交通手段の完全単体化とExcel連携
- `form.html` の交通手段セレクトボックスについて、「航空機・バス」「電車・車」の選択肢を廃止し、**「航空機」「バス」「電車」「自家用車」の4つの単体選択肢**に変更すること。
- `ExcelExportService.java` を修正し、エクセルの該当セルに、選択された交通手段（例：「自家用車」なら「自家用車( 100 )㎞」のように）正しく印字されるようにすること。

## ⑥ CCへの追加指示（GitHub Push）
- 実装完了後、ローカルへのコミットだけでなく、**必ず `git push origin main` を実行する**ようCCに指示すること。
- また、CCのKazumax向け完了報告（チャット）の中で「GitHubへプッシュしました」と一言添えるように指示すること。
