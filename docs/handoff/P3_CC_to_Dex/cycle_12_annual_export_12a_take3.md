## 修正内容

Dex指摘の通り、原本のマージセル構造を確認したところ以下でした。

- `S16:X16`（成年男子・金額欄）/ `S17:X17`（少年男子・金額欄）
- `AB16:AE16`（成年女子・ラベル）/ `AB17:AE17`（少年女子・ラベル）
- `AF16:AK16`（成年女子・**金額欄**）/ `AF17:AK17`（少年女子・**金額欄**）
- `AL16`（単位「円」・固定文字列）/ `AL17`（同）

Take2の実装は女性側を`AL`列（列インデックス37）へ書き込んでおり、本来の金額欄`AF`列（列インデックス31）ではなく、単位「円」の固定文字列セルを数値で上書きしてしまっていました。

`writeBreakdownPair`の女性側書込列を`37(AL)`から`31(AF)`へ修正し、コメントも実態に合わせました。

```java
writeSafeNumeric(sheet, topRow0idx, 31, adultFemale);      // AF列（成年女子、AF:AK結合の金額欄。AL列は単位「円」のため書き込み禁止）
writeSafeNumeric(sheet, topRow0idx + 1, 31, youthFemale);  // AF列 次行（少年女子、AF:AK結合の金額欄）
```
