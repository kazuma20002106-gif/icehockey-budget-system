## Take3 NGの真因（Take3報告の誤りのお詫びを含む）

Take3 P4レビューで、`AGENTS.md` / `CLAUDE.md` / `.cursorrules` からTake2で追加したはずのルールが再び消えていると指摘されました。実際にファイルを直接確認したところ、ご指摘の通り**Take2の追記内容が跡形もなく消え、汎用テンプレートそのものの内容に戻っていました**。Take3報告で「Take2から変更なし」と書いたのは誤りでした。

### 原因調査の結果

デスクトップの `マニュアル同期.ps1` を確認したところ、次の仕様であることが分かりました。

- `AI共通マニュアルテンプレ` フォルダの `AGENTS.md` / `CLAUDE.md` / `.cursorrules` / `manuals/` を、登録された複数プロジェクト（`シフト`, `budget-system`）へ **`Copy-Item -Force` で無条件上書き**する。
- テンプレート側は一度も更新されておらず、Take2でCCが直接 `budget-system` 側の3ファイルに書き込んだ内容は、テンプレートには一切反映されていなかった。
- この状態で同期スクリプトが再実行されると、`budget-system` 側の3ファイルはテンプレートの内容（＝Take2以前の汎用状態）で無条件に上書きされ、Take2の追記は跡形もなく失われる。

`AGENTS.md` 自身に「`AGENTS.md`、`CLAUDE.md`、`.cursorrules` だけを直接変更して終わってはいけない。共通ルールの本物は常に `AI共通マニュアルテンプレ`」と明記されている通り、**この3ファイルへの直接パッチはそもそも同期に対して脆弱な設計**でした。Take2・Take3はこの設計上の弱点を認識せずに直接パッチを当てていたため、同じ問題が再発しました。

## Take4での対応方針（変更点）

今回はTake2/Take3と異なり、**`AGENTS.md` / `CLAUDE.md` / `.cursorrules` には budget-system 固有の具体的ルールを直接書き込みません**。同期で消えない場所に本体を置き、同期される入口ファイルからはそこを参照させる構成に変更しました。

### 1. 共通テンプレート（`AI共通マニュアルテンプレ`）を汎用的に強化

以下は budget-system 固有の内容を含まない、全プロジェクト共通の改善のため、テンプレート本体に追加しました。

- `docs/PROJECT_RULES.md` / `docs/handoff/WORKFLOW_RULES.md` が存在する場合は「拘束力のある必読ファイル」であると明記（従来は「存在すれば読む」という弱い表現だった）。
- `docs/PROJECT_RULES.md` に書かれうる内容の例（Airの事前宣言、危険タスクプロセス、バージョン管理、Dexの重点レビュー・保存先）を一般化して明記。
- CC向けに「完了時の報告ルール」（P3保存先、合図文の単独コードブロック化）を追加。
- Dex向けに「P2/P4の出力ルール」（保存先、チャット短文化、合図文コードブロック化）を追加。
- **新設「共通ルールと同期に関する重要事項」**：3ファイルはマニュアル同期の対象であり、プロジェクト固有の内容を直接書くと同期で消えることを明記。今回の事故の再発防止です。

### 2. budget-system固有ルールは `docs/PROJECT_RULES.md`（同期対象外）に集約

新設「0. Air / CC / Dex の追加ルール（このプロジェクト固有）」セクションに、Take2/Take3で `AGENTS.md` 等に直接書いていた内容をすべて移設しました。

- Airの事前宣言・Kazumax承認待ちルール、危険タスクの完全プロセス
- CCの `app.version` 更新、`.\mvnw.cmd -q -DskipTests compile`、`[vX.Y.Z]` コミット規約
- Dexの P2/P4保存先、保存処理・DB・Excel出力・金額計算・参加者管理・日付計算の重点レビュー

このファイルの冒頭に「配置理由」として、同期スクリプトによる上書き事故が実際に起きたことを明記し、以後この設計を崩さないよう注記しています。

### 3. `マニュアル同期.ps1` に安全策を追加

- 上書き対象ファイルの内容がテンプレートと異なる場合（＝プロジェクト側で直接編集された可能性がある場合）、**必ずタイムスタンプ付きバックアップを作成**してから上書きするように変更。
- 同期完了後、差分があったファイルの一覧と各バックアップパスを `Write-Warning` で明示するように変更。
- 実際に本スクリプトを実行し、`シフト` と `budget-system` の両方で警告とバックアップが正しく機能することを確認済みです（バックアップ先は各プロジェクトの `docs/manual_legacy/`）。

### 4. 同期の実行と反映確認

`AGENTS.md` の「共通マニュアル更新ルール」の手順（テンプレート更新 → 同期実行 → 反映確認 → 報告）に従い、実際に `マニュアル同期.ps1` を実行しました。

- `budget-system` の `AGENTS.md` / `CLAUDE.md` / `.cursorrules` が新テンプレート内容に更新されたことを確認済み。
- `シフト` 側にも同じ汎用強化が反映されました（hockey/budget-system固有の内容は含まれていないため、無関係な混入はありません）。
- 上書き前の内容は両プロジェクトとも `docs/manual_legacy/*.diverged-backup` に保存済みです。
