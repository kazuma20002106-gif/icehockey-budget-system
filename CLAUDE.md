# 開発における絶対ルール (Git & バージョン管理)

1. 作業（タスク）が一つ完了するごとに、必ず変更内容を Git にコミット（commit）すること。
2. コミットする際は、必ずコミットメッセージの先頭にバージョン表記（例: `[v1.0.1] 〇〇の機能を追加`）を義務付けること。
3. コミット後、必ずリモートリポジトリ（GitHub等）へプッシュ（push）すること。
4. 今後、Claude Code が作業を行う際は、毎回必ずこのルールに従って自律的に保存とプッシュを完了させてから「作業完了しました」とユーザーに報告すること。

---

## 補足（このルールを守るための実務メモ）

- **リモート**: `origin` = https://github.com/kazuma20002106-gif/icehockey-budget-system.git
- **対象ブランチ**: `main`
- **バージョンの管理場所**: `src/main/resources/application.properties` の `app.version`。
  コミット時はこの値とコミットメッセージ先頭の `[vX.Y.Z]` を一致させて更新する。
- **バージョン採番の目安（セマンティックバージョニング）**:
  - パッチ（`v1.6.6 → v1.6.7`）= バグ修正・小さな調整
  - マイナー（`v1.6.x → v1.7.0`）= 機能追加
  - メジャー（`v1.x → v2.0.0`）= 大きな仕様変更・互換性のない変更
- **標準手順**:
  1. `application.properties` の `app.version` を更新
  2. `git add -A`
  3. `git commit -m "[vX.Y.Z] 変更内容の要約"`
  4. `git push origin main`
- コミット本文の末尾には共著表記を付与する（`Co-Authored-By: Claude ...`）。
