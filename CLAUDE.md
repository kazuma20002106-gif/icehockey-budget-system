# Claude Code Instructions

このファイルは CC（Claude Code）向けの入口です。

## 最初に読むもの

作業前に必ずプロジェクト直下の `AGENTS.md` を読み、続けて以下を確認してください。

1. `docs/handoff/CURRENT_STATUS.md`
2. `manuals/AI_TEAM_WORKFLOW.md`
3. `manuals/WORKFLOW_RULES.md`
4. `docs/PROJECT_RULES.md`（存在する場合・必読）
5. `docs/handoff/WORKFLOW_RULES.md`（存在する場合・必読）

## CCの基本役割

- AirまたはDexが作成した実装指示書に従って実装してください。
- 通常タスクでは、AirのP1/P2統合指示書がそのままCC向け指示書になる場合があります。
- 危険タスクでは、Dexが事前監査したP2指示書に従ってください。
- 指示書が見つからない場合は、勝手に推測で実装せず、現在の状況を確認してください。
- 作業完了後は、変更内容と検証結果を `docs/handoff/P3_CC_to_Dex/` など、プロジェクトで指定されたhandoff先へ保存してください。

## Gitとバージョン管理

このテンプレートでは、特定の言語、フレームワーク、GitHub URL、バージョンファイルは指定しません。
コード変更時のバージョン管理・ビルド・検証コマンド・コミット規約は、`docs/PROJECT_RULES.md` がある場合は必ずそちらに従ってください。このファイル自体に具体的なコマンドを書き込まないでください（理由は下記「共通ルールと同期に関する重要事項」を参照）。

## 完了時の報告ルール

- P3報告書は `docs/handoff/P3_CC_to_Dex/` に保存すること。
- Kazumaxへのチャットは短くまとめ、次担当への合図文は必ず単独のコードブロック（テキストボックス）で出すこと。地の文に埋め込まない。

## 共通ルールと同期に関する重要事項

`AGENTS.md`、`CLAUDE.md`、`.cursorrules` はこのプロジェクト専用ファイルではなく、`マニュアル同期.ps1` によって複数プロジェクトへ配布される共通テンプレートです。プロジェクト固有のビルドコマンドやバージョン管理ルールをこのファイルに直接書き込むと、同期の再実行で無条件に上書きされ消えます。必ず `docs/PROJECT_RULES.md` など同期対象外のファイルに書いてください。

## 禁止事項

- 別プロジェクトのリモートURL、バージョン番号、サイクル番号を使わないでください。
- プロジェクト固有のビルドコマンドが不明なまま、推測で危険なコマンドを実行しないでください。
- ユーザーや他AIが作った未確認の変更を勝手に戻さないでください。
